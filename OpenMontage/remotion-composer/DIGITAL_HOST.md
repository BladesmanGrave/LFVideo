# 数字人（VRM 主播）— 现状与路线

数字人是一套**帧确定性**的 VRM 主播系统：全身只规范化渲染一次，每个场景再用命名预设对这张渲染做 2D 裁切 / 落点 / 缩放，配合暖色调色与右上角外框，让 2.5D 角色和平面 UI 融为一体。所有动作、口型、取景都由 `useCurrentFrame()` 驱动，可复现。日常出 JSON 只需选 `cut.type`，取景会自动配好；需要特殊处理时再写一行覆盖。用法详见 `SCENE_TYPES.md`。

## 已完成（main）

- **#1 场景化取景预设系统**：全身只渲一次，命名预设（角落小头像 / 左右半身 / 左右全身 / 主持人 / 隐藏）做 2D 后裁切；`cut.type → 预设` 自动映射，cut 之间约 9 帧缓动平滑过渡。解析优先级 `cut.avatar > avatar.byType > 内置映射 > avatar.default > 兜底`。
- **#2 暖色 anime-ghibli 风格**：调色板 / 背景 / 玻璃面板 / 开场标题全转暖，叠加全局 soft-light 暖调色 + 暗角，解决「冷场景 vs 暖角色」的违和感（不上 Unity）。
- **#3 圆形 / 圆角矩形外框**：新增 `corner-circle-tr` / `corner-rounded-tr` 两个右上角「徽章」外框（描边 + 柔光底 + 投影，cover 居中），按需通过 `cut.avatar` 启用。

## 后续推荐

1. **实拍接入**：挑 ep02 几个密集信息场景试用右上角圆形外框，确认整集观感后再决定是否设为某些 `cut.type` 的默认。
2. **随取景自适应的接触阴影**：#2 暂未带入脚下阴影（旧版是给固定全身面板做的，会随动态裁切错位）；可改为根据当前 layout 的落点/缩放动态绘制，让全身预设真正「站」在画面里。
3. **整集成片复核**：在暖色 + 新取景下重渲一版完整 ep02，作为风格定稿前的最终校验。
4. **（可选）更沉浸**：若仍想要真 3D 景深 / 角色站进场景，再评估 Unity 离线预渲染场景当背景层的方案（解耦渲染、保住帧确定性）。
5. **多角色 / 预设可视化调参**：后续要支持多个角色模型或给运营做取景微调 UI 时再排期。
